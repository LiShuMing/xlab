# 虚拟内存与页表

从问题出发：

一次malloc/new从用户态到内核态到底经历了什么？有几次syscall？TLB发生在哪个阶段？页表的访问在哪个阶段？ 页表同申请内存大小有什么关系？

```cpp
void* p = malloc(10);
```

一次申请内存的完整时序图：
```
时间轴 →
│
├─ T0: 程序执行 new/malloc
│
├─ T1: 触发系统调用 (mmap/brk)
│      └─ CPU: Ring 3 → Ring 0
│      └─ MMU: 切换页表基址
│
├─ T2: 内核创建VMA，建立页表项(PTE Present=0)
│
├─ T3: 返回用户态，程序获得虚拟地址
│
├─ T4: 程序首次访问该地址
│      └─ MMU查TLB → 未命中
│      └─ MMU查页表 → PTE Present=0
│      └─ 触发页错误异常 (#PF)
│
├─ T5: 内核页错误处理程序
│      └─ 分配物理页框 (struct page)
│      └─ 更新PTE (Present=1, 物理页框号)
│      └─ 刷新TLB
│
├─ T6: 重新执行访问指令
│      └─ MMU查TLB/页表 → 命中
│      └─ 访问物理内存
│
└─ T7: 对象创建完成
```

┌─────────────────────────────────────────────────────────────────────────┐
│  各阶段耗时估算 (x86_64, 现代CPU)                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  阶段      操作                      耗时 (纳秒)        说明            │
│  ─────────────────────────────────────────────────────────────────      │
│  T0        malloc检查bins            10-50             用户态          │
│  T1        系统调用进入              200-500           Ring切换        │
│  T2        创建VMA+PTE               500-1000          内核分配        │
│  T3        返回用户态                200-500           Ring切换        │
│  T4        首次访问+页错误           50-100            到异常触发      │
│  T5        页错误处理+物理分配       1000-10000        最耗时!         │
│  T6        重试访问                  1-5               TLB命中         │
│  ─────────────────────────────────────────────────────────────────      │
│  总计 (首次访问)                   ~2000-12000 ns     微秒级          │
│  后续访问                          ~1-5 ns            纳秒级          │
│                                                                         │
│  关键洞察: 首次访问比后续访问慢 1000-10000 倍!                           │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
