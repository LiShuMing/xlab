## Problem Statement

用户需要一个高效的邮件管理工具，当前系统存在以下问题：

1. **展示内容零散** — 无法查看邮件原文
2. **界面不友好** — 热门话题、今日话题等分散注意力
3. **缺乏状态追踪** — 无法区分已读/未读邮件
4. **无自动化** — 需要手动同步，旧邮件不会自动清理

## Goals

- 单列收件箱界面，以邮件为核心
- 支持查看原文（点击展开/收起）
- 追踪 Gmail 未读状态 + 系统处理状态
- 自动同步最近 N 天邮件
- 自动清理过期邮件

## Non-Goals

- 项目发现功能（暂不保留）
- 话题追踪功能（暂不保留）
- 日报生成功能（暂不保留）
- 发送邮件功能

## Data Model

### messages 表（全新设计）

```sql
CREATE TABLE IF NOT EXISTS messages (
    id           TEXT PRIMARY KEY,      -- Gmail message ID
    thread_id    TEXT NOT NULL,
    subject      TEXT,
    sender       TEXT,
    sender_email TEXT,                  -- 提取出的邮箱地址
    received_at  TEXT NOT NULL,         -- ISO-8601 UTC
    body_text    TEXT,                  -- 邮件原文（存储期内保留）
    msg_state    TEXT NOT NULL DEFAULT 'unread',  -- 'unread', 'read', 'starred'
    relevance    TEXT,                  -- 'high', 'medium', 'low', 'skip'
    summary_json TEXT,                  -- AI 摘要
    created_at   TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ','now'))
);

CREATE INDEX idx_messages_received ON messages(received_at);
CREATE INDEX idx_messages_state ON messages(msg_state);
CREATE INDEX idx_messages_relevance ON messages(relevance);
```

**msg_state 状态：**
- `'unread'` — Gmail 未读 / 未处理
- `'read'` — Gmail 已读 / 已处理
- `'starred'` — 标星（可扩展）

### settings 表

```sql
CREATE TABLE IF NOT EXISTS settings (
    key TEXT PRIMARY KEY,
    value TEXT NOT NULL
);
```

**默认设置：**
- `retention_days`: 7
- `auto_sync_interval_minutes`: 30

### REST API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/messages` | 邮件列表 |
| GET | `/api/messages/{id}` | 邮件详情（含原文） |
| PATCH | `/api/messages/{id}` | 更新邮件状态 |
| POST | `/api/sync` | 手动触发同步 |
| GET | `/api/settings` | 获取设置 |
| PUT | `/api/settings` | 更新设置 |

### GET /api/messages

**查询参数：**
- `state`: `unread` | `read` | `all`（默认 `all`）
- `relevance`: `high` | `medium` | `low`
- `days`: 最近 N 天（默认使用 retention_days 配置）
- `page`: 页码（默认 1）
- `size`: 每页数量（默认 50）

**响应：**
```json
{
  "total": 42,
  "unread_count": 5,
  "high_relevance_count": 8,
  "messages": [
    {
      "id": "msg-xxx",
      "subject": "Re: Apache Iceberg",
      "sender": "user@apache.org",
      "received_at": "2026-03-24T10:00:00Z",
      "msg_state": "unread",
      "relevance": "high",
      "summary": "AI 生成的摘要..."
    }
  ]
}
```

## Frontend Design

### 收件箱主页面

**布局：**
- 顶部：标题 + 设置按钮 + 同步按钮
- 筛选栏：状态筛选、重要性筛选、时间范围
- 统计栏：未读数、高优先数
- 邮件列表：卡片式，点击展开原文

**排序逻辑：**
1. 未读邮件（msg_state = 'unread'）
2. 高优先邮件（relevance = 'high'）
3. 其他邮件
4. 各自按 received_at 倒序

**展开原文交互：**
- 默认显示摘要
- 点击「展开原文」按钮，显示 body_text
- 点击「收起原文」按钮，隐藏原文

### 设置页面

**可配置项：**
- 邮件保留天数（retention_days）
- 自动同步间隔（auto_sync_interval_minutes）

## Background Tasks

### 启动流程

```
Server 启动
    │
    ├── 1. 初始化数据库
    │
    ├── 2. 启动后台同步任务
    │       │
    │       ├── 拉取最近 N 天邮件
    │       ├── 同步 msg_state
    │       └── 对新邮件调用 AI 摘要
    │
    └── 3. 启动定时清理任务
            │
            └── 每日凌晨删除过期邮件
```

### 增量同步

```python
def sync_messages():
    last_history_id = get_last_history_id()
    new_messages = gmail_api.fetch_since(last_history_id)

    for msg in new_messages:
        save_message(msg)
        summary = llm_summarize(msg.body_text)
        save_summary(msg.id, summary)

    unread_ids = gmail_api.get_unread_ids()
    update_msg_state(unread_ids)
```

### 自动清理

```python
def cleanup_old_messages():
    retention_days = get_setting('retention_days', default=7)
    cutoff_date = today - timedelta(days=retention_days)

    deleted = db.execute(
        "DELETE FROM messages WHERE received_at < ?",
        (cutoff_date,)
    )
```

## Risks & Mitigations

| 风险 | 缓解措施 |
|------|----------|
| Gmail API 限流 | 使用增量同步，避免全量拉取 |
| AI 摘要超时 | 异步处理，先存原文后补摘要 |
| 数据丢失 | 保留 Gmail 原始邮件，只删除本地 |
