Apache Iceberg Primary Key Index Position Deletes Flink CDC Spark MERGE INTO Secondary Index Framework
Huaxin Gao provided an update on the indexing work for the Iceberg table format, indicating a strategic shift from bloom filter indexes to primary key indexes. This decision was driven by ambiguity regarding whether bloom filters fit within the secondary index framework or should be classified as extended stats. The new proposal outlines a primary key reverse-lookup index that maps each key to its specific physical location, including file path and row position. This implementation enables scan-time file pruning specifically optimized for point lookups. Additionally, it allows for the conversion of key-based deletes into position deletes, effectively eliminating equality deletes for Flink CDC pipelines. The index also accelerates Spark MERGE INTO operations by replacing expensive full-table joins with direct file lookups. A detailed proposal document has been shared with the community for review and feedback.
